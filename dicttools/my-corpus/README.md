# 独自コーパス構成ツールReadme

このディレクトリは、独自のコーパス(文字利用頻度)データベースを構築するためのツールを含む。

## 入力

- source.txt
独自コーパスの基礎情報となる、任意のテキストデータ。
UTF8で記述されていることを前提とする。

- make.corpus.db.sh
上記のsource.txtを各漢字の使用状況を得て、頻度情報(freq)を求める。
本スクリプトを実行することで、corpus.dbが得られる。


- ../jmdict/dict.db (ファイル名は暫定)
辞書データベース。jmdictで処理済のものを利用する

## 中間出力

- *.m4 *.sql *.freq *.base
いずれも中間生成物であり、自動成成される。

## 出力

- corpus.db

独自コーパスを元に求めた使用頻度(freq)のdb。

## コメント

基本情報としてUTF8のテキストファイル、source.txt を準備すれば、独自コーパスを用いた、「自家用文字使用ひん度」を辞書に反映できる。

筆者は自身のメルマガ(約5MB)を利用し、KANJIDIC2.xmlのfreq情報を0.7、独自情報を0.3の割合で使用順をアレンジした。

現在、デフォルトで配布している「ひともじ」の、辞書はこれをベースとしている。

なお、独自コーパスがない場合でも、KANJIDIC2.xmlのfreq情報を用いて辞書構築は可能。

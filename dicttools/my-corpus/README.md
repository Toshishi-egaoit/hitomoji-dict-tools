# my-corpus

独自コーパスから文字利用頻度 DB を構築するための任意フェーズです。

配布リポジトリには、個人コーパスの本文データは含めません。
利用する場合は、各環境で `source.txt` を用意します。

## 入力

- `source.txt`
  独自コーパスの基礎情報となる任意のテキストデータ。
  UTF-8 で記述されていることを前提とします。

- `../kdic2/dict-kdic2.db`
  文字と UCS コードの対応を得るために参照します。
  別の DB を参照する場合は `DICT_DB=/path/to/dict.db` を指定します。

## 実行

```sh
make -C ../kdic2
make
```

## 中間出力

- `corpus.base`
- `corpus.freq`
- `corpus.m4`
- `freq.m4`
- `corpus.sql`

いずれも中間生成物であり、自動生成されます。

## 出力

- `corpus.db`
  独自コーパスを元に求めた文字使用頻度 DB。

## コメント

`source.txt` を用意すれば、独自コーパスによる自家用の文字使用頻度を
辞書に反映できます。

筆者は自身のメルマガ(約5MB)を利用し、KANJIDIC2.xmlのfreq情報を0.7、独自情報を0.3の割合で使用順をアレンジした。

独自コーパスがない場合でも、KANJIDIC2 の freq 情報を用いて辞書構築は可能です。

## 削除

生成物を削除する場合は、トップディレクトリから次を実行します。

```sh
make clean-corpus
```

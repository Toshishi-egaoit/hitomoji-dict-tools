# jmdict

JMdict を使った読みの補完と除外を扱うフェーズ群です。

ここでは、KANJIDIC2 だけでは不足する読みを補い、逆に KANJIDIC2 に含まれるが
現代語での使用例が確認しにくい読みを除外します。

## フェーズ

- `p1/`: 使用頻度が高く、踏み外しがないという前提で自動登録する。
- `p2/`: `p1` 適用後、頻度が低い読みや踏み外しの可能性が高い読みを人手で選別して追加登録する。
- `ex/`: KANJIDIC2 にあり、JMdict に使用例がない読みを除外する。

## 入力

- `../archive/JMdict.xml`

`archive/` は上流データの置き場であり、Git 管理対象には含めません。

## 生成物

- `jmdict.db`

JMdict XML から抽出した熟語表記と読みの突合用 DB です。

## 方針

- 機械生成の候補と、人手で判断した結果を分けて管理します。
- 人手で判断したファイルは、辞書を再現するための正式な入力として扱います。
- 最終的な統合と `hitomoji.dic` の生成は `../build/` 側で扱います。

## ツール

- `dictmatch.cpp`
  KANJIDIC2 由来の読みと JMdict の熟語読みを突合する。

## 移行方針

- `dictmatch.cpp` は JMdict 突合専用のため、このディレクトリで管理する。
- `p1/`, `p2/`, `ex/` の各フェーズから共通ツールとして利用する。

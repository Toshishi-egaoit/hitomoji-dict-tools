# 日本語IME「ひともじ」専用辞書ツール

このリポジトリは、日本語IME「ひともじ」用の漢字辞書を作成するためのツール群を提供しています。
日本語IME「ひともじ」については、以下をごらんください。
https://github.com/toshishi-egaoit/hitomoji/

日本語IME「ひともじ」はWindowsで動作しますが、このリポジトリ内のツール群はいずれも、Unix(Linux)での利用を前提としています。

このリポジトリは、ひともじの辞書を自作したり、改造したりする際に必要となるものです。
「ひともじ」の利用を目的とする方には不要なツール群です。

このリポジトリには、辞書データは含んでいません。
ひともじの標準辞書は、KANJIDIC2プロジェクトの成果物(KANJIDIC2.xml)を利用しています。

本リポジトリに含むツールは以下の通りです。

* slot_greed:各漢字の配置をGreed法で決めるためのツール
* dictgen:DB(SQLite3)に構築されたデータを「ひともじ」専用のバイナリ形式に変換するツール
* dictview:辞書ビューア
* dictedit:辞書編集ツール(v0.3では未実装)

本リポジトリでは、以下のツール/ライブラリを使用しています。

* SQLite3のライブラリ(libsqlite3.lib)
* g++
* xmlstarlet
* その他、unixの標準ツール(bash/sed/m4など)

このリポジトリのバージョンは、hitomojiリポジトリにあわせているため、v0.3からはじまっています 。

# ロードマップ

### v0.3.1

目的：漢字データベースの整備

- 各種のコーパスなどから、漢字情報を集められるかどうかを判断する。
- よみ情報としてKANJIDIC2を利用する
- 教育漢字のデータベースを作成する。
- 常用漢字のデータベースを作成する。
- SQLiteを用いたデータベースを構築する
- よみ辞書の最低限の整備を行う
-- よみ辞書の音/訓の識別を行う。
-- よみ辞書に音便(だく音化、そく音化)を考りょした読みを追加。
-- 現状で、約2100の漢字と7000のよみデータがDBに登録されている。

### v0.3.2

目的：漢字をキーボードにマッピングするしくみの検討

- 漢字の優先順位は、KANJIDICに含まれる使用頻度の情報をベースとする。
- 自身のメルマガ(がんばりすぎないセキュリティ.テキストで約6MB)の漢字使用状況をデータ化する。
- 両方の使用率を合成(KANJIDIC 0.7/メルマガ0.3)して、文字の優先順位を決める。
- 優先度を利用し、greed法でキー位置(スロット)のマッピングツールを作成。

### v0.3.3

目的：辞書の構造の検討と辞書生成ツールの作成

- 辞書のファイル形式の検討→バイナリ形式とする。
- SQLiteのデータベースから、バイナリファイルを生成するツール(dictgen)の作成
- 作成したバイナリファイルの動作確認のため、一覧表示ツール(dictview)の作成

## v0.3.4

目的：辞書メンテツールの作成

- SQLiteのDBを更新するツールを作成
- コマンドライン形式で、よみや文字の追加を行うことができる。
- コマンドは以下の通り
- add kanji
- add yomi
- remove kanji
- remove yomi
- パラメタは未定。
- ただし、実装はv0.4以降で行う(時期は未定)
